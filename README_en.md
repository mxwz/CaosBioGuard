## 🧭 System Portal

This system adopts a completely **Cloud-Edge decoupled** architecture. To provide the clearest guidance, we have split the documentation into two dedicated independent modules. Please proceed to the corresponding documentation based on your needs:

### ☁️ [Web Admin Server ➔](web_admin/README_en.md)
The "Brain" of the system. Provides a global visual dashboard, personnel feature orchestration, attendance/access log analysis, and remote device parameter configuration.
- Architecture: `Flask` + `MySQL` + `Jinja2`
- Scenarios: Cloud servers, local data centers

### 👁️ [Edge AI Device ➔](sideUI/README_en.md)
The "Sensory" of the system. Deployed at turnstiles or physical entrances, providing millisecond-level offline face recognition, anti-spoofing interception, real-time voice broadcasting, and asynchronous offline caching.
- Architecture: `PySide6` + `InsightFace` + `PyTorch` + `SQLite`
- Scenarios: Windows PC, x86 hosts with cameras, Raspberry Pi

## 🔗 Cloud-Edge Collaborative Architecture

The system is deeply optimized in data synchronization and storage, ensuring stable operation even in weak network environments.

- **🗄️ Hybrid Storage Model**: The edge adopts `SQLite` + local image hash caching for millisecond-level authentication response; the server relies on `MySQL` for massive data persistence.
- **📡 Bi-directional Sync**:
  - **Uplink**: Attendance records, access logs, and liveness detection alarms generated by the edge are smoothly uploaded to the server via background threads.
  - **Downlink**: Configuration changes, newly registered faces, admin permissions, and soft-delete instructions issued by the server are pushed to corresponding edge devices in real-time.
- **🛡️ Data Isolation & Security**: Introduces a `device_id` mechanism where different edge devices only pull facial features and configurations belonging to them, achieving physical isolation of terminal data.

## 🛠️ Environment & Dependencies

This project relies on `PyTorch` for model inference:
- **GPU Version (Recommended)**: Please visit the [official PyTorch website (https://pytorch.org/)](https://pytorch.org/) to manually install the GPU version for optimal performance.
- **CPU Version**: If you are using a pure CPU environment, please manually install the CPU version of `onnx` and the CPU version of `PyTorch`, otherwise you may encounter runtime errors.

> 💡 **Edge Database Configuration Note**:
> When configuring the MySQL connection on the Edge device (SideUI), **DO NOT use** `127.0.0.1` or `localhost` as the server address (this points to the edge device itself). You MUST use the server's actual **LAN IP** or **Public IP** so the edge device can correctly connect to the server's database across the network.
